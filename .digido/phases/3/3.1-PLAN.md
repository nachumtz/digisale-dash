# Phase 3.1: Error Handling + Logging

**מטרה:** לעטוף את כל נקודות הכשל האפשריות ב-try/except, להציג הודעות ידידותיות, ולרשום שגיאות טכניות ל-`error_log.md`.

**קבצים מעורבים:**
- `error_logger.py` — חדש
- `data_loader.py` — עדכון
- `app.py` — עדכון

**Tasks:**
- [ ] צור `error_logger.py` עם פונקציה `log_error(context, error)` שכותבת שורה לקובץ `error_log.md` בפורמט: `[TIMESTAMP] [CONTEXT] ERROR: {error}`
- [ ] עטוף ב-`data_loader.py` את כל פונקציות הטעינה ב-try/except: תפוס `KeyError` (עמודה חסרה), `ValueError`, `ParserError` — זרוק Exception מובנה עם הודעה בעברית
- [ ] ב-`app.py` עטוף את קריאות `data_loader` ב-try/except: בקריסה — `st.error("שגיאה: {msg}")` + קריאה ל-`log_error`

**Success Criteria:**
- העלאת קובץ עם עמודה חסרה → `st.error` ברור, האפליקציה לא קורסת
- קובץ Excel שגוי → הודעה ידידותית, לא traceback
- `error_log.md` נוצר ומתמלא בשגיאות עם timestamp
