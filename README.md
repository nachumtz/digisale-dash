# 📊 Digisale-Dash | דשבורד מכירות

מערכת דשבורד מבוססת Streamlit המציגה נתוני מכירות (B2B/B2C) בצורה אינטראקטיבית ומתקדמת, בפריסת Dark Corporate מלוטשת. המערכת נועדה לניהול ומעקב בזמן אמת אחר קטגוריות, סגמנטים, ורווחיות של חברת Digisale, ומאפשרת סינון נתונים מהיר ברמת העיר.

## 🌟 פיצ'רים מרכזיים
- עיבוד נתונים מהיר המשלב מידע ממקורות שונים (הזמנות, לקוחות, מוצרים).
- מנגנון העלאת קבצי CSV/Excel בצורה מאובטחת ומקומית (ללא העלאה לשרת חיצוני).
- חיווי ויזואלי מהיר (KPIs) של פדיון, רווח, ביצוע הזמנות ואחוזי ביטול.
- סינון (Filter) בזמן אמת לפי **עיר**.
- גרפים אינטראקטיביים מדויקים (Plotly CSS Dark Theme) לפילוח הכנסות לפי קטגוריות וסגמנטי לקוחות. 
- ניהול שגיאות (Error Handling) ידידותי למשתמש עם קובץ תיעוד טכני בלוגים.

---

## 💻 דרישות מערכת
- Python גרסה 3.9 ומעלה (3.9+).
- אין צורך בחיבור אינטרנט פעיל בזמן הריצה (המערכת רצה מקומית, Localhost).
- מערכת הפעלה נתמכת: Windows, macOS, Linux (בסביבת Virtual Environment).

---

## 🚀 הוראות התקנה והרצה (ב-5 דקות)

1. **שכפול הפרויקט (Clone)**
   ```bash
   git clone <REPOSITORY_URL>
   cd digisale-dash
   ```

2. **יצירת סביבה וירטואלית (Virtual Environment)**
   ```bash
   python -m venv venv
   ```

3. **הפעלת הסביבה הוירטואלית**
   - ב-Windows (PowerShell):
     ```bash
     .\venv\Scripts\Activate.ps1
     ```
   - ב-Mac/Linux:
     ```bash
     source venv/bin/activate
     ```

4. **התקנת תלויות (Dependencies)**
   ```bash
   pip install -r requirements.txt
   ```

5. **הרצת האפליקציה**
   ```bash
   streamlit run app.py
   ```
   *האפליקציה תיפתח אוטומטית בדפדפן בכתובת `http://localhost:8501/`.*

## 📋 דרישות מבנה נתונים (קלט)

המערכת מצפה לקבל ב-Sidebar שלושה קבצים התואמים למבנה הבא. על הקבצים מסוג CSV/XLSX להכיל את עמודות החובה (רגיש לאותיות גדולות/קטנות):

| שם הקובץ המצופה | עמודות חובה | משמעות/הערות |
|---|---|---|
| **Orders** (הזמנות) | `Order_ID`, `Customer_ID`, `Product_ID`, `Order_Date`, `Quantity`, `Status` | `Status` חייב להכיל `"הושלם"` כדי להיחשב ברווח/פדיון. |
| **Customers** (לקוחות) | `Customer_ID`, `Customer_Segment`, `City` | ללא ערכים אלו הגרפים ואפשרויות הסינון יכשלו. |
| **Products** (מוצרים) | `Product_ID`, `Category`, `Unit_Price`, `Cost_Price` | נדרש לחישוב הרווח (Unit Price - Cost Price) וההכנסות. |

## 🛠️ ניפוי שגיאות (Troubleshooting)
במידה והאפליקציה מזהה קובץ שבור או חוסר בעמודה חיונית:
1. המערכת **לא תקרוס**, אלא תציג הודעת שגיאה מסודרת על המסך (בתוך ה-Streamlit).
2. פרטי השגיאה המלאים, כולל חותמת זמן, נרשמים טכנית בתוך קובץ ה-**`error_log.md`** אשר נמצא בשורש הפרויקט לבקרה וטיפול.
